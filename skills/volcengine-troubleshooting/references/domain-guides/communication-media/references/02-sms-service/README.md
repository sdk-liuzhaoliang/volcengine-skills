# 短信服务

用于短信发送失败、回执失败、签名/模板/号码资质审核问题。

## 前置输入

- 手机号脱敏样本、签名、模板 ID、子账号、发送时间、RequestId。
- 是接口失败、平台发送失败，还是运营商回执失败。

## 先判断方向

| 现象 | 优先检查 |
|---|---|
| `SY:0500` | 平台侧受理、签名/模板/余额/频控；不要直接归为最终回执失败 |
| `RE:0500` | 运营商回执、号码侧状态、内容或资质 |
| 资质审核未通过 | 号码资质、签名、模板审核 |
| `AccessDenied` | 转账号权限 skill |

## 查询能力

当前 `cli-meta` 未在公共 CLI 中明确匹配短信服务；不要编造 `ve sms ...`。

若需要只读事实，按官方旧版 Python SDK `volcengine.sms` 的映射处理：

- `get_sms_send_details`
- `get_signature_and_order_list`
- `get_signature_ident_list`
- `get_sms_template_and_order_list`
- `get_sub_account_detail`

## 关注字段

- 发送明细、回执码、失败阶段。
- 签名、模板、资质审核状态。
- 子账号、模板与签名是否属于同一业务上下文。

## 结果解读

| 证据 | 常见结论 |
|---|---|
| 接口成功但回执失败 | 不是 SDK 调用成功就代表最终送达 |
| 模板/签名待审核或驳回 | 先处理审核，不继续归因网络 |
| 权限失败 | 保留短信 Action，转账号权限 |
| 余额或额度异常 | 转计费 skill |
