# 高频 Playbook

## 短信 `SY:0500`

1. 先确认是平台受理阶段失败。
2. 查签名、模板、资质、余额、频控。
3. 不把它直接解释成运营商回执失败。
4. 看到该错误码时优先读取本 skill，不先直接走通用文档搜索。

## 短信 `RE:0500`

1. 先确认接口是否已经成功受理。
2. 再看回执与号码侧状态。
3. 如果用户只给了接口成功结果，仍需继续问回执。

## RTC `ERROR_CODE_INVALID_TOKEN`

1. 保留 AppID / room / uid。
2. 查 token 过期和签发上下文一致性。
3. 只有控制面 Action 报错时才转 OpenAPI skill。

## VOD `GetPlayInfo`

1. 先查 Vid 和发布状态。
2. 再查期望清晰度/格式是否有产物。
3. 地址能返回但播不了时，转 CDN/域名/播放器方向。

## Live 推流失败

1. 先确认推流 URL 与鉴权。
2. 再确认流是否进入平台。
3. 不能把拉流问题和推流问题混成一个结论。

## CV/OCR 异步任务失败

1. 先确认 submit 是否成功。
2. 再查 task_id、输入 URL、任务状态。
3. `Bad request` 优先按参数问题处理。
